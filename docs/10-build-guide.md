### 4.12 SharePoint フォルダ権限設定

> 前提: SP サイトの管理者権限を持っていること
> ACL 検証のために、テストサイトのフォルダに異なる閲覧権限を設定する。
> **実施タイミング**: §4.0〜4.11 の Azure リソース構築完了後、インジェスト（§4.13）の直前に実施する。

**A. 権限設計（個人名ベース）**

> セキュリティグループは使用せず、個人ユーザーに直接権限を付与する。
> `app-sprag-poc` のインジェストはアプリケーション権限（`Sites.Read.All`）で動作するため、SP のフォルダ権限とは独立。

| フォルダ | 閲覧可 | 閲覧不可（ACL テスト検証用） |
|---------|--------|----------------------|
| `01_ダミーファイル 経営` | PJリーダー氏 | 構築担当者氏・メンバーA氏・他エンジニア |
| `02_ダミーファイル 人事労務` | 全員（権限継承のまま） | — |
| `03_ダミーファイル 営業` | PJリーダー氏・メンバーA氏 | 構築担当者氏・他エンジニア |

> **注意**: 構築担当者氏は構築担当だが、インジェストはアプリ権限で動作するため SP 閲覧権限は不要。権限を絞ることで構築担当者氏自身も ACL テスト対象にできる。テストケース作成時に各フォルダの中身を確認する必要があるため、権限変更はリソース構築完了後・インジェスト直前に行う。

**B. 作業タイミング**

| フェーズ | 作業 | 構築担当者氏の権限 |
|---------|------|------------|
| §4.0〜4.11 Azure リソース構築 | リソース作成に集中 | 全フォルダ閲覧可（現状維持） |
| リソース構築完了後 | SP フォルダの中身を確認しながらテストケースを作成 | 全フォルダ閲覧可（現状維持） |
| テストケース確定後 | 本セクション C〜E の SP 権限変更を実施 | 02_人事労務のみ閲覧可に変更 |
| §4.13 インジェスト実行 | 権限情報を含めて文書取り込み | 変更後の状態（アプリ権限で取り込むため影響なし） |
| ACL テスト | テストケースに沿って検証 | 01_経営・03_営業が見えないことを検証可能 |
